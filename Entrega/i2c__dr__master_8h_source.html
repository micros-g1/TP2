29 /**
30  * @brief I2C initialize master mode register configuration.
31  * @details Initializes master internal configuration on the i2c module.
32  * Sets a callback for a specific I2C module
33  * to be called whenever a hardware interrupt that is related to this master is called.
34  * Check the MK64 user manual (Table 51-5) for information on what hardware interrupts would be called
35  * and the status, flag and enable they signify.
36  * Calling the init function twice has no effect (safe init).
37  * @param mod : I2C module the callback is referring to.
38  * @param callback : callback to be called when a hardware interrupt is called.
39  */

69 /**
70  * @brief I2C Get Data Transfer Complete
71  * @details Gets the current status of the last data transfer.
72  * This call is valid only during or immediately following a transfer to or from the I2C module.
73  * The Transfer status is cleared by reading the I2C data register in receive mode
74  * or by writing to the I2C data register in transmit mode.

78 /**
79  * @brief I2C Get the bus status
80  * @details I2C Gets the bus status: Busy or not.
81  * The bus may be busy when this or some other master is taking over it

83  * @return *true* when an interrupt is pending, *false* otherwise.

119 /**
