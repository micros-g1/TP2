Detailed Description

I2C Master Interface.

Author
Grupo 1 Labo de Micros
Date
28 Sep 2019

This interface is prepared to handle communications using any I2C module as a master in a non-blocking way. there are many unhandled cases (this is not a generic i2c master interface as it does not handle multimaster!!) Although not all multimaster cases are handled, some of them may be implemented for future upgrades/implementations.

◆ i2c_master_int_bus_busy()

bool i2c_master_int_bus_busy ( i2c_module_id_int_t  mod_id)

I2C Master BUS is busy.

Get the current status of the I2C BUS for a specific I2C module ( Busy or not) The Bus is busy when every START signal sent to the bus has encountered its corresponding STOP signal in the bus (both sent by this master module).

Parameters
mod_id: I2C module for which the bus status will be checked.
Returns
true is the bus is currently busy. false otherwise.

)

I2C Master Read Data.

sends a read data package from a specific I2C module. If specified, the read package will be preceded by a write package with information (for the slave) about the type of reading that will be performed. If this package is not needed, then it will not be sent.

Parameters
mod_id: I2C module from which data will be read.
question: reading information transferred (with a write frame) to the slave before the reading sequence (Repeated Start + Address + Read) is sent.
len_question: amount of bytes question has. If 0, then no write package will be sent.
amount_of_bytes: amount of bytes to be received when the reading is performed.
