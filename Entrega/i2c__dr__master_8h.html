Detailed Description

I2C Master Driver.

Author
Grupo 1 Labo de Micros
Date
28 Sep 2019

I2C Driver This driver should be used for the pin control of I2C master modules of the MK64F. there are many unhandled cases (this is not a generic i2c master driver as it does not handle multimaster!!)

◆ i2c_dr_bus_busy()

bool i2c_dr_bus_busy ( i2c_modules_dr_t  mod)

I2C BUS is busy.

Check the status of the bus BUSY flag. IMPORTANT : This flag does not update immediately after a start signal has been been detected on the bus, so, in case the user wants to keep updated in real on the state of the bus, he/she/it/Apache Helicopter should implement a software layer that either sets a small delay after sending a start signal or solves this problem in any other way.

Parameters
mod: I2C module to get the bus status from.
Returns
true if the bus is busy. false otherwise.

bool i2c_dr_get_transfer_complete ( i2c_modules_dr_t  mod)

I2C Get Data Transfer Complete.

Gets the current status of the last data transfer. This call is valid only during or immediately following a transfer to or from the I2C module. The Transfer status is cleared by reading the I2C data register in receive mode or by writing to the I2C data register in transmit mode.

Returns
false when the last data transfer is still in progress. true when completed.

◆ i2c_dr_master_init()

void i2c_dr_master_init ( i2c_modules_dr_t  mod,
i2c_service_callback_t  callback 
)

I2C initialize master mode register configuration.

Initializes master internal configuration on the i2c module. Sets a callback for a specific I2C module to be called whenever a hardware interrupt that is related to this master is called. Check the MK64 user manual (Table 51-5) for information on what hardware interrupts would be called and the status, flag and enable they signify. Calling the init function twice has no effect (safe init).

Parameters
mod: I2C module the callback is referring to.
callback: callback to be called when a hardware interrupt is called.

bool  enabled 
)

I2C Enable or Disable Start and Stop Interrupts.

Disables or enables the start or stop detection interrupts. NOTE: To clear the I2C bus stop or start detection interrupt: In the interrupt service routine, first call i2c_dr_clear_startf, and then call i2c_dr_clear_iicif. If this sequence is reversed, the IICIF flag is asserted again.

Parameters
enabled: true when the start stop interrupts should be enabled. False otherwise.
